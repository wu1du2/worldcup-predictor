## 本轮改动

- 临时策略搜索池改为按策略族均衡采样，而不是直接取前 N 个策略。这样可以避免泊松 EV、trend 等后置策略被前面的生成器遮住。
- 稳定型从普通平局锚点升级为限赔平局锚点。旧的高 ROI 平局变体如果依赖超高赔率长尾命中，会被拒绝。
- 共识型从纯低赔共识升级为“共识 + 泊松 EV”。它带来小幅正 ROI，同时提升了推荐解释质量。
- 价值型保留健康的 3 注泊松 EV 变体。分数更高的 1 注变体仍被拒绝，因为平均下注数低于 1.5，对面向用户的推荐来说波动过大。
- 2026-06-28 第二轮假设把实验池扩到 546 个，并把回测采样上限放宽到覆盖全部候选。价值型升级为“泊松 EV + 赛果方向多样性”两格，避免候选全部挤在相邻低平局。
- 2026-06-28 第三轮先试“平局锚点 + 双向一球胜负”，失败：命中增加但成本过高，最好总分只有 64。随后试“多样性 EV + 1-1 低赔保护”，成功：价值型升级到 76.2，命中从 8/56 提到 13/56，平均下注仍保持 2.5。
- 2026-06-28 第四轮先试“只在 λ 接近时触发 1-1 保护”，失败：当前 context 期望进球差异不敏感，多数候选与 active 等价。随后改用盘口阈值，把 1-1 保护从 7 放宽到 7.5，成功：价值型升级到 77.7，ROI +26.06%，命中 14/56。
- 2026-06-28 第五轮继续尝试稳定型和共识型：稳定型改为“1-1/0-0 + 两个最低赔非平局”，ROI 提升到 +15.94%，平均下注降到 2.4；共识型改为“外部来源 + 6 倍内低赔 + context_v1 泊松补位”3 格，总分提升到 71.1，语义也更贴近共识型。
- 2026-06-28 第六轮尝试三个新假设，并修复前端迭代图的版本承接：稳定型新增“只有热门方向明确才加单边保护”，代表候选 `tem_draw_anchor_directional_homeaway2_draw5_5_gap3_cap25` 总分 72.2、ROI +16.22%，但低于现役稳定型 74.1；价值型新增“1-1/0-0 相对赔率结构”候选 `tem_poisson_drawstructure_context_v3_n2_ratio1_35_cap35_p0_006`，总分 74.5、ROI +23.5%，仍低于现役价值型 77.7；共识型尝试低赔 cap 5.5，ROI 转负。结论：不升级生产 router，但前端保留 v4 失败实验，让平台期可见。
- 2026-06-28 第七轮把临时策略回测上限从 700 提到 1200，修复新增候选没有真正进入 loop 的问题。本轮新增四组候选：更细 draw-guard 阈值、强弱修正平局锚点、Poisson + 低赔共识保护、热门方向小胜保护。结果没有任何候选超过现役三旗舰；最佳稳定型仍是 `tem_draw_anchor_lean_homeaway2_draw6_cap22`，总分 74.9，但未超过 +1 晋级门槛；最佳价值型仍是现役 drawguard 7.5。
- 2026-06-29 自动 loop 正式跑通：`npm run strategy:knockout-auto` 连续跑 6 轮，第 6 轮触发“连续 5 轮最高分提升 < 0.5”的平台期停止条件。每轮都写入后台 `strategy_loop` 报告，前端“后台报告”可以看到中文摘要和详细停止原因。

## 当前平台期 case

- 稳定型：省注版本牺牲 1 场命中，但 ROI、形态评分和平均下注数都更健康。继续增加 2-2 或第三个非平局会抬高成本。
- 价值型：1 注泊松变体 ROI 更高，但平均下注数过低，仍按不健康单点策略拒绝。当前 active 的 draw-guard 7.5 版本同时提升 ROI 和命中，最大命中赔率仍为 28。
- 共识型：4 格版本命中更高但形态略差；3 格版本综合分更高、成本更干净。纯低赔共识命中更多，但解释性和收益都更弱。
- 第六轮新假设：方向触发稳定型和相对平局结构价值型都有正收益，但综合分没有超过现役；低 cap 来源共识压缩过度，反而削弱收益。
- 第七轮新假设：强弱悬殊时删除 0-0 会显著伤害 ROI；Poisson 后加通用低赔共识没有击败 drawguard；热门方向小胜保护更符合直觉，但历史 proxy 里成本增加大于补回收益。

## 下一轮搜索假设

- 稳定型：不要继续只靠“加/删比分格子”搜索，下一步应让盘口强弱、总进球倾向和来源 context 共同决定是否触发非平局保护。
- 价值型：下一轮重点改概率模型或 EV 修正项，而不是继续扩大 drawguard 阈值；可加入市场热度、外部来源 over/under、BTTS 倾向作为 λ 或 draw multiplier 修正。
- 共识型：继续保留 3 格结构。下一轮应把外部来源按可信度/一致性打分，而不是简单把明确比分和低赔共识拼接。
- 前端迭代图：`knockoutStrategyEvolutionBuilder` 的 base versions 必须包含当前已上线 active 版本，否则下一轮实验会反复显示成 v3，而不是继续追加 v4/v5。
